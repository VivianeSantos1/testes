Consultar versão
git --version
Configurar GIT
git config --global user.name "wcomandoschools-test"
git config --global user.email "test@wcomandoschools.com"
Iniciar um novo repositório
git init
Verifica se o arquivo já está no repositório
git status

Nota:

TRACKED --> é quando o arquivo está no github
UNTRACKED --> é quando o arquivo está apenas localmente no seu dispositivo

Adiciona ao ambiente de testes e preparação (STAGE)
git add
Adicionando vários arquivos ao ambiente de teste
git add --all ou git add -a
O commit serve como um save point, ou seja a versão
finalmente adicionada ao repositório, o commit deve
ser acompanhado de uma mensagem descritiva
git commit -m "First release of Hello World!"
Mostra o status com descrições reduzidas
git status --short
M index.html
NOTA: Os sinalizadores de status curtos são:

?? - Arquivos não rastreados
A - Arquivos adicionados ao stage/palco
M - Arquivos modificados
D - Arquivos excluídos

Enviando diretamente, ignorando o envio ao STAGE
git commit -a -m "Updated index.html with a new line"
Apresenta histórico de modificações do repositório
git log
Associando repositório do github ao git
git remote add origin https://github.com/user/repo.git
Subindo os arquivos locais já existentes
git push --set-upstream origin master
É possível consultar uma ajuda no própio git com os comandos:
git (comando) -help ou git help --all (este exibirá uma lista muito longa)
